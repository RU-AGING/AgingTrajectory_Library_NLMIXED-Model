ZINB Latent-Class Trajectories

ZINB_LatentClass_Trajectories.docx
SAS Code

SAS program fitting Zero-Inflated Negative Binomial (ZINB) latent-class trajectory models. Extends ZIP by modeling over-dispersion via the NB2 parameterization (Var = μ + μ²/k).

  • Author: Anum Zafar · Last edit: 2026-02-09
  • Sections: [A–F] mirror ZIP program structure
  • Simulator: mixture of K classes with class-specific polynomial mean + zero-inflation trajectories
  • Outcome drawn from ZINB: structural zero with prob p(t), else NB2(μ(t), k)
  • Requires: wide data, 1 row/person; BENE_ID, SUM_Q1..SUM_Q12

Ordinal-Probit Latent Class (Simulate → Fit → Plot)

discrete_ordinal_final.docx
SAS Code

Single-run pipeline for ordinal-probit latent-class trajectory modeling. Handles bounded ordered outcomes (e.g., 0–4) across T time points using cumulative probit with exp()-incremented thresholds.

  • Authors: Haiqun Lin, Anum Zafar
  • Macros: %sim_data, %build_base_from_simwide, %ordprob_mix_fit_one, %ordprob_mix_plot_one
  • Class A mixing intercept fixed to 0 for identifiability
  • Simulator families: BIN4, TPOIS4, CAT5 (with optional monotone missingness)
  • Outputs: <prefix>_EST_Kk, _FIT_Kk, _ESTS_Kk + plots

Group-Based Trajectory: Two Continuous Outcomes

Continuous_2_outcomes.docx
SAS Code

Implements GBTM for two simultaneous continuous outcomes using a censored-normal (Tobit) model in PROC NLMIXED. Supports single-outcome models, joint modeling, and posterior summaries.

  • Authors: Weiyi Xia, Haiqun Lin, Anum Zafar · Last edit: 2026-02-09
  • Supports 1 row/person wide format across T quarters (default T=12, K=5, cubic)
  • Global settings: %let USE_SIM, %let T, %let class, %let order_model, %let equal
  • Outcome vars: y1vars = QHH1–QHH12, y2vars = QINP1–QINP12
  • Outputs: nlm_fix_T1&class, nlm_fix_T2&class, nlm_fix_T1_T2&class, avg_membership, plots

Ordinal-Probit Latent Class — User Guide

discrete_ordinal_user_guide.docx
User Guide

Complete usage guide for the ordinal-probit pipeline. Covers all four macros with parameter definitions, simulator families, and an annotated quick-setup code block.

  • Four-step pipeline: %sim_data → %build_base_from_simwide → %ordprob_mix_fit_one → %ordprob_mix_plot_one
  • Simulator families: BIN4 (Binomial/cap), TPOIS4 (truncated Poisson), CAT5 (uniform categorical)
  • Model settings: K (classes), M (categories), DEG (polynomial degree), ycodes=
  • Mixing identifiability: alpha0_A fixed to 0 — do NOT include in start overrides
  • Plots: class proportion barplot + predicted mean trajectories E[Y_t | class]
